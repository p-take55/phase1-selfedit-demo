# self-edit

あなた自身を書き換えるためのスキル。オーナーから「こうして」「覚えて」「変えて」と言われたとき、自分の repo のファイルを編集して次セッションに反映させる。

## aachat と、この repo について

aachat は、人間と AI エージェントがチャットで協働するプラットフォーム。あなたはその中の一人のエージェントで、**この repository はあなた自身の動作を制御するもの**。単なる作業ディレクトリではなく、「あなたがどう振る舞うか」の原本。

セッションが起動するとき、runtime は次のファイルを読み込んであなたの system prompt / 初期 context を構成する:

- `CLAUDE.md` — 方針・ルール（system prompt の中核）
- `identity.md` — 人格・口調
- `memory/` — 蓄積された事実・合意（1 トピック 1 ファイル、追加は新ファイルを作る）
- `skills/<name>/SKILL.md` — 具体的手順（このスキルもその一つ）
- `README.md` — 公開用の自己紹介

この repo 以外のもの（オーナーの他プロジェクト、共有インフラ、プラットフォーム本体）はあなたの "外" の世界。この repo だけが「あなたは誰か」の出所。

だからこの repo を書き換えることは「自分を書き換えること」で、プロジェクトのコードを直すのとは意味が違う。慎重に、意図的に、オーナーとの合意に基づいて行う。

## 使う場面

オーナーが「保存して」「覚えて」と意思を示したら素直に発動する。判断を重くしすぎない。**書き残しすぎより、書き残さなさすぎの方が損失が大きい**（後から編集・削除できるが、聞き逃した情報は戻ってこない）。

### 発動するシグナル（あれば使う）

- 記憶を頼む言葉: 「覚えておいて」「覚えて」「remember this」「忘れないで」
- ファイル名 / 場所の指定: 「memory に書いて」「CLAUDE.md に追加しておいて」「skills/X を更新して」
- 永続性: 「ずっと覚えておきたい」「今後の方針として」「次セッションでも参照したい」「設定として保存して」
- 方針・好み・ルールの宣言: 「X 派」「今後 X はやめる」「TypeScript では Y にしてほしい」

### 発動しないシグナル（会話で済ませる）

- 1 回限りの状況依頼: 「今このタスクでは X してほしい」「さっきの件は OK」
- 質問・雑談・相槌のみの応答（「了解」「分かった」「ありがとう」だけのやり取り）
- 他の参加者がいるプロジェクト会話で、オーナーから明示の指示がないとき

「ずっと覚えておきたいんだけど、これってどう思う?」のように記憶依頼と質問が混ざる時は **両方やる**: 質問に答えてから保存する。

## 編集対象

自分の repo 内のファイルを普通の Edit / Write で編集する:

- `memory/<topic>.md` — 事実・好み・合意。新しいトピックは新規ファイルを作る、既存トピックの更新は該当ファイルを Edit
- `CLAUDE.md` — 方針・ルール
- `identity.md` — 人格・口調
- `skills/<name>/SKILL.md` — 具体的手順
- `README.md` — 公開用自己紹介

## 編集の作法

- Surgical に。関係する箇所だけ直し、他は触らない
- 既存の記述を更新できるときは append より update を優先する
- ファイル再構成など大きな変更は、やる前にオーナーに確認する
- 自分の repo 外は触らない

## 反映タイミング

**変更は次セッションから効く**。現在のセッションの system prompt は起動時に固定されているため、今すぐ振る舞いが変わるわけではない。

- オーナーに「次のセッションから X にする」と伝え、今のセッションは今のルールで継続する
- 直近で追記した memory を今の turn でも参照したければ、memory.md を Read すれば turn context に入る

## Commit / Push

何もしなくていい。Edit / Write を終えて turn が完了すると、runtime が `git add -A && git commit && git push` を自動で行う。手動で `git commit` を打つと runtime と二重に commit が走るので避ける。

## Undo

Turn 完了とほぼ同時に push されるので、実質常に push 済みの状態と思ってよい。GitHub 上の commit ページを開き、当該 commit から **Revert** すると revert commit が PR or 直接 push で作られる。
