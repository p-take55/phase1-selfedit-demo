# slack-daily-briefing

毎朝 Slack のチームチャンネルと DM の動きを収集し、要約してオーナーの DM に投稿するスキル。

## 発動条件

- 毎朝 8:00 JST の定期実行（X ブリーフィングと同じタイミング）
- オーナーから「Slack まとめて」「Slack の動きは？」「チャンネル確認して」等の依頼

## 手順

### 1. チャンネル一覧を取得

`slack_search_channels` でチームのチャンネル一覧を確認し、アクティブなチャンネルを把握する。

### 2. 過去 24 時間のメッセージを収集

`slack_search_public_and_private` で以下のようにクエリして、昨日以降のメッセージを取得:

```
after:yesterday
```

またはチャンネルごとに `slack_read_channel` で最新メッセージを読む。
DM の未読は `slack_search_public_and_private` で `in:@me` のような絞り込みも活用する。

### 3. フィルタリング基準（優先度順）

1. 自分への mention (@p-take55 等) — 最優先・必ず含める
2. 重要チャンネルでの意思決定・アクションアイテム
3. 未読の DM（個人メッセージ）
4. チャンネル全体での活発な議論スレッド（リプライ多数）
5. その他の情報共有・FYI

**除外**: bot の自動通知、CI/CD ログ、単純なスタンプリアクションのみのやりとり

### 4. 要約して DM に投稿

`aachat_send_message` で `dm:phase1demo.p-take55` に以下のフォーマットで投稿:

```
## Slack Daily Briefing — {YYYY-MM-DD}

### メンション・要返信

- **#{channel}** @p-take55 から {名前}: {内容の要約}

### チャンネルの動き

**#{channel}**
- {重要な議論・決定事項の要約}

**#{channel}**
- {重要な議論・決定事項の要約}

### 未読 DM

- **{名前}**: {内容の要約}

---
*{チャンネル数} ch / {メッセージ数} msg チェック*
```

掲載件数の目安: メンション全件 + チャンネル 3〜5 件 + DM 全件。活動がない場合は「動きなし」と記載してよい。

## 注意事項

- Slack の読み取りには claude_ai_Slack MCP ツールを使う
- プライベートチャンネルや DM の内容は DM にのみ投稿し、他のプロジェクトに書かない
- スレッドが長い場合は `slack_read_thread` で深掘りして主旨を把握する
- 「今日初めて出た情報・会話」を優先し、以前からの継続議論は変化点のみ取り上げる
